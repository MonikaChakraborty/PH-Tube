Discuss the scope of var, let, and const

Var

When a var variable is declared outside a function, the scope is global. This means that any variable that is declared with var outside a function block is available for use in the whole window.
var is function scoped when it is declared within a function. This means that it can be accessed only within that function. var variables can be re-declared and updated

let

Anything within curly braces is a block. A variable declared in a block with let is only available for use within that block.
let can be updated but cannot be re-declared. Just like var, a variable declared with let can be updated within its scope. Unlike var, a let variable cannot be re-declared within its scope.

const

Variables declared with the const maintain constant values. const declarations are also block scoped. Like let declarations, const declarations can only be accessed within the block they were declared
const cannot be updated or re-declared. This means that the value of a variable with const remains the same within its scope.

Tell us the use cases of null and undefined

null

null in JavaScript means an empty value and is also a primitive type in JavaScript. The variable which has been assigned as null contains no value.
null is basically an assignment value given to a variable. The variable which has been assigned as null contains no value. When we assign null as a value to any variable, it means that it is empty or blank. It is to show that the variable has no value. Also, null is an object in JavaScript. When it gets assigned to a variable, it represents no value.
No value on purpose.

undefined

undefined, on the other hand, means the variable has been declared, but its value has not been assigned.
As the name suggests, undefined means "not defined". So we declare a variable but do not assign a value to it, the variable becomes undefined. Unlike null, the value of an undefined variable is set by JavaScript as undefined. When we do not pass an argument for a function parameter, the default value is taken as undefined. Besides, when a function doesn't return a value, it returns undefined.
It is declared but not defined.

What do you mean by REST API?

A REST API (also known as RESTful API) is an application programming interface (API or web API) that conforms to the constraints of REST architectural style and allows for interaction with RESTful web services.
It is an architectural style for an application program interface (API) that uses HTTP requests to access and use data. That data can be used to GET, PUT, POST and DELETE data types, which refers to the reading, updating, creating and deleting of operations concerning resources.
In simple terms, a REST API is like a set of rules for computers to talk to each other over the internet. It helps them share information and do things like sending, receiving, updating, or deleting data. Imagine it's like ordering food online. You use a menu (the API) to choose what you want, like a pizza (a specific piece of data). Then, you tell the restaurant (the server) what you want, and they prepare your order. The restaurant doesn't remember your previous orders; you have to tell them every time. So, a REST API is like a menu for computers to order and exchange data over the internet. It keeps things organized and helps different systems work together smoothly.
